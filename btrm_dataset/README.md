# futudiffu-btrm: Sparse-Step Diffusion Trajectory Dataset

Sparse-step diffusion trajectory data for training Bradley-Terry reward models (BTRM) on the Z-Image NextDiT architecture. Each trajectory captures intermediate latent states at selected steps during the denoising process, enabling reward models to learn preferences over generation quality as a function of step count and attention quantization.

## Dataset Description

This dataset contains 50 diffusion trajectories generated by a custom FP8 blockwise NextDiT inference pipeline. Trajectories are saved at sparse step indices rather than every step, reducing storage while retaining the key transition points in the denoising process.

The dataset supports two discrimination tasks for BTRM training:

- **Scrongle pairs**: Same seed and prompt, different step count. The reward model learns to distinguish full-step (30-step) gold-standard outputs from reduced-step variants.
- **Scrimble pairs**: Same seed, prompt, and step count, different attention backend. The reward model learns to detect quantization artifacts introduced by INT8 attention (SageAttention) compared to full-precision SDPA.

## Data Format

### Directory structure

```
btrm_dataset/
    manifest.json                     # Generation schedule and trajectory index
    latents/
        traj_000000/
            meta.json                 # Generation parameters for this trajectory
            step_00.pt                # Latent at step index 0
            step_04.pt                # Latent at step index 4
            step_09.pt                # Latent at step index 9
            step_14.pt                # Latent at step index 14
            step_19.pt                # Latent at step index 19
            step_24.pt                # Latent at step index 24
            step_29.pt                # Latent at step index 29
            final.pt                  # Final latent (post inverse noise scaling)
        traj_000001/
            ...
```

### Latent tensors

- **Format**: PyTorch `.pt` files (saved with `torch.save`)
- **Shape**: `(1, 16, H, W)` bfloat16
- **Standard t2i resolution**: 1280x832 pixels, yielding latent shape `(1, 16, 104, 160)`
- **i2i resolutions vary**: 496x544, 512x512, 832x1280 depending on the source image

### Sparse step indices

Full 30-step trajectories save latents at indices `[0, 4, 9, 14, 19, 24, 29]` plus the final latent (8 tensors total). Reduced-step trajectories save at whichever of these indices fall within their step range, plus the final latent.

### Trajectory metadata (meta.json)

Each trajectory's `meta.json` contains:

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | `"t2i"` (text-to-image) or `"i2i"` (image-to-image) |
| `seed` | int | PRNG seed for noise generation |
| `prompt` | string | Full text prompt |
| `prompt_idx` | int | Index into the 24-prompt template library (t2i only) |
| `n_steps` | int | Total Euler sampling steps |
| `precision` | string | Attention backend: `"sdpa"` or `"sage"` |
| `batch_idx` | int | Generation schedule batch index |
| `denoise` | float | Denoise strength, i2i only (range 0.75-0.95) |
| `image_file` | string | Source image filename, i2i only |
| `output_width` | int | Output width in pixels, i2i only |
| `output_height` | int | Output height in pixels, i2i only |

### Step count distribution

The first 20 trajectories (batch 0 and 1) use 30 steps with both SDPA and SageAttention backends. Batches 2 and 3 use reduced step counts (8-22 steps) to provide scrongle training signal.

## Usage

### Loading a single trajectory

```python
import json
import torch
from pathlib import Path

traj_dir = Path("btrm_dataset/latents/traj_000000")

# Load metadata
with open(traj_dir / "meta.json") as f:
    meta = json.load(f)

# Load a specific step latent
step_04 = torch.load(traj_dir / "step_04.pt", weights_only=True)
# shape: (1, 16, 104, 160), dtype: torch.bfloat16

# Load the final latent
final = torch.load(traj_dir / "final.pt", weights_only=True)
```

### Loading all sparse steps for a trajectory

```python
SPARSE_INDICES = [0, 4, 9, 14, 19, 24, 29]

latents = {}
for idx in SPARSE_INDICES:
    path = traj_dir / f"step_{idx:02d}.pt"
    if path.exists():
        latents[idx] = torch.load(path, weights_only=True)

final = torch.load(traj_dir / "final.pt", weights_only=True)
```

### Iterating over the manifest

```python
with open("btrm_dataset/manifest.json") as f:
    manifest = json.load(f)

for record in manifest["records"]:
    print(f"{record['type']} | steps={record['n_steps']} | "
          f"attn={record['precision']} | {record['prompt'][:60]}...")
```

## Generation Details

Trajectories were generated using the futudiffu inference server with the following configuration:

- **Diffusion model**: Z-Image NextDiT, FP8 blockwise quantized (~5.8 GB)
- **Text encoder**: Qwen3-4B, bfloat16 (~7.5 GB)
- **Sampling**: Euler discrete schedule, CFG scale 4.0, sampling shift 1.0
- **Attention backends**: PyTorch SDPA (bfloat16, full precision) and SageAttention (INT8 QK + BF16 PV)
- **Hardware**: NVIDIA RTX 4090 (SM 8.9, Ada Lovelace)
- **Precision**: All bfloat16 throughout the pipeline (text encoder, diffusion backbone, sampling loop, noise, latents, sigmas)

For i2i trajectories, source images are forward-noised to a configurable denoise level (0.75-0.95 range under the flow-matching schedule), then denoised with a text query composed of an object label (describing the source image content) and a transformative style label.

## Intended Use

This dataset is designed for:

1. **BTRM reward model training**: Training a Bradley-Terry preference head on top of the Z-Image backbone to score trajectory quality. The BTRM head uses LoRA adapters on the last 2 transformer layers with a Linear(3840, 2) output head producing two logits (one per discrimination task).

2. **REINFORCE policy optimization**: Using the trained BTRM as a reward signal for policy gradient optimization with sparse step sampling (3-5 steps per 30-step rollout).

3. **Attention quantization analysis**: Comparing SDPA vs. SageAttention trajectories to characterize and detect INT8 quantization artifacts in latent space.

This is an early-stage dataset (50 trajectories). The full generation plan targets ~2304 trajectories across 100 seeds, 24 prompts, 8 step schedules, 2 attention backends, and 11 i2i reference images.

## Limitations

- The dataset is small (50 trajectories) and intended as a development/validation set, not a production training corpus.
- Latents are stored as `.pt` files using `torch.save` (pickle-based). A v2 format using safetensors and parquet indexing is planned.
- Cross-session reproducibility is approximate (cosine similarity ~0.999 but not bitwise identical) due to non-deterministic CUDA operations.
- All trajectories were generated on a single RTX 4090. Cross-architecture equivalence (e.g., H100) has not been validated for this dataset.

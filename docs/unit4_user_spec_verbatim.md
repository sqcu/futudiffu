# Unit 4 User Specification (Verbatim)

new unit 4 is "PINKIFY and THISNOTTHAT". PINKIFY is a proper BTRM or BTRM-analogue; there is a literal ranking rule (PINKER) which says the pinker of two images is the 'preference' for any two images; the rule can be implemented in any manner but i suggest something using a prenorm and a 'definition of the pink' wherein images which have a contrast between a locally pink vs locally nonpink color and higher total volumes of the pink-by-contrast tone are the PINKER winners. THISSER means, if we resize and rescale the THIS image ('/mnt/f/dox/repos/ai/futudiffu/i2i_off_policies/pizza-ratto.png') and THAT image ('/mnt/f/dox/repos/ai/futudiffu/i2i_off_policies/offhand_pleometric.png') to the image we are comparing, the image is THISSER if the similarity of the THIS image and the judged image is higher than the similarity of the THAT image and the judged image. this *too* is a pairwise comparison: images which are more similar to THIS than they are to THAT get higher scores, creating two different 'score adjustment fields', in effect: one which nudges all images with no semblance of THAT towards THIS, and another which nudges all images similar to THAT and not THIS *away* from looking like THAT. crucially, both of these 'known literal rule pairwise preferences' could be consumed as direct rewards for RL methods *or* used to train a BTRM model along these pairwise distinctions. we want to end to end test that we can train a RL model against these pairwise comparisons in isolation from any RL process that consumes the RL model! so the standalone demonstration of the PINKIFY, THISNOTTHAT heads forms another end to end test (we can literally train a BTRM head, which is persistable, loadable, bit-for-bit comparable across persist and load, can be applied to score distributions of noisy latents and denoised latents, (since we train the BTRM adapter against a diffusion model which *reads* noisy latents, and both of our judgements, even if about pixel definitions of images, do not need to be differentiable through the vae decoder (even though by coincidence they might be) to train a BTRM model against the scalar relative judgements they entail).
